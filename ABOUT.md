## 🧠 Core Idea

Instead of repeatedly solving the same problems, PULSE leverages historical incident data to:

- Suggest the most relevant fixes  
- Reduce debugging time  
- Assist on-call engineers in real time  

---

## 🖥️ Features

### 📊 Pulse Dashboard (Command Center)
- Provides a real-time overview of system health  
- Displays AI learning activity  
- Serves as a central hub for engineers and responders  

### 📁 Incident History
- Stores all previously reported errors  
- Shows how each issue was resolved  
- Enables faster debugging through historical insights  

### 📝 Report Incident

Incidents can be reported in two ways:

- **Manual Input:** Enter error details directly  
- **Webhook Integration:** Connect external tools such as Datadog, Sentry, and PagerDuty  

After submission, the AI analyzes the issue and suggests the most suitable fix based on past data.

---

## ⚙️ How It Works

1. An incident is reported  
2. The system searches for similar past logs  
3. The AI analyzes patterns and context  
4. The most relevant solution is suggested  
5. The system improves continuously through feedback  

---

## 🚀 Why This Project?

- Reduces repeated debugging effort  
- Acts as a centralized team memory system  
- Assists during high-pressure incident scenarios  
- Improves over time as a self-learning system  

---

## 🛠️ Tech Stack

- **Frontend:** React, Tailwind CSS  
- **Backend:** Node.js, Express  
- **Database:** Firebase  
- **AI Layer:** LLM + HindSight Cloud (for memory storage)  

---

## 📌 Future Improvements

- Advanced ranking (BM25 / vector search)  
- Automated incident clustering  
- Real-time alert prioritization  
- Multi-team collaboration features
